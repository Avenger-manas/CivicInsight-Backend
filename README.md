CivicInsight-Backend: The Core Command Engine ⚙️This is the central orchestrator for CivicInsight, built using Spring Boot. It manages the business logic, database transactions, automated workflows (SLA), and acts as the bridge between the Frontend and the Ollama-powered AI Engine.🏗️ Architecture OverviewThe backend follows a clean, modular architecture:Controller Layer: REST Endpoints for Citizen, Officer, and Admin dashboards.Service Layer: Contains the "Brain" for Auto-assignment, SLA logic, and Escalation.Repository Layer: MongoDB integration for high-speed, flexible data handling.AI Integration: A Feign Client/RestTemplate that connects to the local Llama 3 instance.🛠️ Tech StackFramework: Spring Boot 3.2.xLanguage: Java 17+Database: MongoDB (NoSQL for flexible grievance schemas)Security: Spring Security with JWT (Role-Based Access Control)Scheduling: Spring Scheduler (For real-time SLA monitoring)AI Bridge: Python Flask API (connecting to Ollama)🚀 Key Backend Logic1. Smart Dispatcher (Auto-Assignment)When a grievance is submitted, the backend calculates a priority score and assigns it to the most relevant officer.Criteria: Department Match + City/Ward Match + Current Workload (Active Tickets).2. SLA & Escalation EngineA background cron job runs every hour to check for overdue tickets:High Priority: Must be addressed in 24 hours.Medium/Low: 48-72 hours.Action: If a ticket is overdue, the escalationLevel is bumped, and an automated alert is sent to the higher-level Political Admin.📂 Project StructureBashsrc/main/java/com/civicinsight
